# AGENTS.md

Guidance for coding agents working in `Dirstral/homebrew-tap`. This is a parallel
companion to `CLAUDE.md`: both convey the same rules for different agent
platforms. They are kept intentionally near-identical, so if their wording ever
appears to diverge, treat them as complementary rather than contradictory.

## What this is

This is the Homebrew tap for the Dirstral CLI tools — formulas only, no
application source. Users install via:

```sh
brew tap Dirstral/tap
brew install dir2mcp        # lean MCP knowledge server
brew install dir2mcp-full   # same, with a bundled Docling runtime
brew install dirstral       # interactive TUI client
```

## Formulas

| Formula | Source | Maintenance |
|---------|--------|-------------|
| `Formula/dir2mcp.rb` | [dir2mcp](https://github.com/Dirstral/dir2mcp) | **Auto-generated by GoReleaser — do NOT edit** |
| `Formula/dirstral.rb` | [dirstral-cli](https://github.com/Dirstral/dirstral-cli) | **Auto-generated by GoReleaser — do NOT edit** |
| `Formula/dir2mcp-full.rb` | [dir2mcp](https://github.com/Dirstral/dir2mcp) | **Hand-maintained** (custom Docling venv pinning) |

> Note: `dirstral` (the TUI client) was split out of the `dir2mcp` monorepo into
> its own [`dirstral-cli`](https://github.com/Dirstral/dirstral-cli) repo, which
> is now the source of truth for `dirstral` releases. The currently checked-in
> `dirstral.rb` (v0.4.0) predates the split and still points at a `dir2mcp`
> release; later versions come from `dirstral-cli`.

## Golden rule: do not hand-edit auto-generated formulas

`dir2mcp.rb` and `dirstral.rb` carry a `# This file was generated by
GoReleaser. DO NOT EDIT.` header. The source repo's release workflow rewrites
and force-pushes them to this tap automatically (GoReleaser + a
`HOMEBREW_TAP_GITHUB_TOKEN` PAT) on every version tag. Manual edits to their
`version`, `url`, or `sha256` will be overwritten by the next release. To change
them, cut a new release from the source repo.

## The one exception

`dir2mcp-full.rb` has no GoReleaser header and is maintained by hand (it pins an
isolated Python Docling environment). It is the only formula safe to edit
directly here.

## When asked to "bump" or "update a formula"

For `dir2mcp`/`dirstral`: tag a release in the source repo — do not edit files
here. For `dir2mcp-full`: edit the formula here and keep its dependency pins
coherent.
