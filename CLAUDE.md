# CLAUDE.md

## What this is

`Dirstral/homebrew-tap` is the Homebrew tap for the Dirstral CLI tools. It hosts
formulas only — there is no application source code here. Users install from it
with:

```sh
brew tap Dirstral/tap
brew install dir2mcp        # lean MCP knowledge server
brew install dir2mcp-full   # same, with a bundled Docling runtime
brew install dirstral       # interactive TUI client
```

## Formulas

| Formula | Source | Maintenance |
|---------|--------|-------------|
| `Formula/dir2mcp.rb` | [dir2mcp](https://github.com/Dirstral/dir2mcp) | **Auto-generated by GoReleaser — do NOT edit** |
| `Formula/dirstral.rb` | [dir2mcp](https://github.com/Dirstral/dir2mcp) | **Auto-generated by GoReleaser — do NOT edit** |
| `Formula/dir2mcp-full.rb` | [dir2mcp](https://github.com/Dirstral/dir2mcp) | **Hand-maintained** (custom Docling venv pinning) |

## Golden rule: do not hand-edit auto-generated formulas

`dir2mcp.rb` and `dirstral.rb` carry a `# This file was generated by
GoReleaser. DO NOT EDIT.` header. They are rewritten and force-pushed to this
tap automatically by the source repo's release workflow (GoReleaser, using a
`HOMEBREW_TAP_GITHUB_TOKEN` PAT) on every version tag. Any manual edit to the
`version`, `url`, or `sha256` of these files will be silently overwritten by the
next release — so don't make them here. To change them, cut a new release from
the source repo instead.

## The one exception

`dir2mcp-full.rb` has no GoReleaser header and is maintained by hand (it pins an
isolated Python Docling environment). It is the only formula it is appropriate
to edit directly in this repo.

## When asked to "bump dir2mcp" or "update the formula"

For `dir2mcp`/`dirstral`: the answer is almost always to tag a release in the
source repo, not to touch files here. For `dir2mcp-full`: edit the formula here
and keep its dependency pins coherent.
